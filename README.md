# Portable Alarm Clock (ポータブル アラーム クロック)

Windows 11標準の「クロック」アプリのアラーム機能を忠実に再現し、ドラッグ＆ドロップによる自由な並び替え機能を追加した、インストール不要のポータブルアラームアプリです。

## 特徴
- **インストール不要・管理者権限不要**: EXEファイルをダブルクリックするだけで即座に起動します。
- **ドラッグ＆ドロップ並び替え**: アラームカードの空白部分をドラッグすることで、自由な順番に並び替えることができます（Alt+↑/↓キーでの並び替えにも対応）。
- **タスクトレイ常駐**: 閉じる「×」ボタンを押すとタスクトレイに格納され、バックグラウンドでアラームを待機します。
- **完全ローカル動作・超軽量**: ネットワーク通信は一切行わず、Windowsに標準搭載されている「.NET Framework 4.8」を利用するため、追加のランタイムインストールなしで動作します。また、単一EXEのサイズは約540KBと極めて軽量です。

---

## 使用方法

### 1. 起動方法
- `PortableAlarmClock.exe` をダブルクリックして起動します。

### 2. アラームの追加・編集・削除
- **追加**: 右下の「＋」ボタンをクリックして時間、アラーム名、繰り返し曜日、スヌーズ時間を設定し、「保存」を押します。
- **編集**: 一覧のアラームカードをダブルクリックすると編集画面が開きます。
- **削除**: アラームカードにホバーすると表示される「ごみ箱」アイコンをクリックするか、編集画面内の「削除」ボタンから削除できます。

### 3. 並び替え方法
- **マウス操作**: アラームカードの空白部分（トグルスイッチやボタン以外のエリア）をマウスで左クリックし、8ピクセル以上ドラッグして目的の位置で離します。
- **キーボード操作**: `Tab` キーで目的のアラームカードを選択（フォーカス）し、`Alt + ↑` または `Alt + ↓` キーを押すことで、カードを上下に移動できます。
- 並び順は即座に保存され、再起動後も維持されます。

### 4. アラームの鳴動
- 設定時刻になると、画面が最前面に表示され、アラーム音が繰り返し再生されます。
- **スヌーズ**: 設定されたスヌーズ時間（デフォルトは10分）の経過後に再鳴動します。
- **停止**: アラームを停止します。曜日指定のない1回限りのアラームは、停止後に自動的にOFF（無効状態）になります。

### 5. 常駐と完全終了
- 右上の「×」ボタンを押した場合は、アプリ画面のみが非表示になり、タスクトレイに常駐します。常駐中も設定時刻になればアラームが鳴ります。
- タスクトレイアイコンをダブルクリック、または右クリックメニューの「開く」を選択すると、メイン画面が再表示されます。
- アプリを完全に終了させるには、**タスクトレイのアイコンを右クリックし、「終了」を選択**してください。確認メッセージが表示され、「終了する」を選択すると完全終了します。

## 重要な注意点と制限事項

- **アプリ終了時の注意**: アプリを完全に終了している間は、アラームは鳴りません。
- **スリープ・休止状態の制限**: 
  - PCの電源が切れている場合、休止状態、またはシステムによってアプリの実行が完全に停止されている場合は、アラームは鳴りません。
  - **スリープからの復帰時**: PCがスリープから復帰した直後、予定時刻から **10分以内** であれば遅延してアラームが鳴ります。10分を超えている場合は、その回の鳴動は自動的にスキップされます。

## データ保存場所
初回起動時、`PortableAlarmClock.exe` が配置されている同じフォルダー内にデータフォルダーが自動作成されます。

- **アラームデータ**: `<EXE配置フォルダー>\AlarmClockData\alarms.json`
- **ログファイル**: `<EXE配置フォルダー>\AlarmClockData\logs\app-yyyyMMdd.log`

> [!WARNING]
> `alarms.json` が手動変更等で破損した場合、アプリは自動的に破損ファイルを `alarms_broken_backup.json` として退避し、空の状態で安全に起動します。

---

## ライセンス・音源情報
- **音源**: アプリ内に合成WAV波形をメモリ上に生成するプログラムを内蔵しており、外部の音声ファイルやシステム音源の無断コピーは使用していません。
- **依存ライブラリ**: Windows標準API、.NET Framework 4.8 WPF、およびデータシリアライズ用の `System.Text.Json` (Fody/CosturaによりEXE内に静的埋め込み) のみで構築されています。
